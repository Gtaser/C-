# C-项目名称： 潜行游戏
软件环境： Windows 
硬件环境： 通用计算机（支持图形显示）
开发工具： Visual Studio / GCC + SDL2 库

项目描述：
本项目是一个基于 C 语言与 SDL2 开发的 2D 潜行游戏，核心研究并实现了一套具有不完全信息推理能力的游戏 AI 系统。玩家需要在一个由猎人巡逻的关卡中，利用掩体隐藏自己，并最终安全抵达出口。AI 猎人通过有限状态机（FSM）驱动，并采用结合了概率热度图与贝叶斯更新思想的搜索算法，使其行为在具有挑战性的同时，又避免了模式化与死板，显著提升了游戏的策略深度与沉浸感。

核心玩法：

潜行与躲避：玩家使用 WASD 键在关卡中移动，必须避开猎人的视野与巡逻路径。

掩体利用：场景中设有灌木、衣柜等掩体。玩家进入掩体后将完全隐身，AI 无法感知；但离开掩体时会短暂暴露，需谨慎选择时机。

目标与失败：玩家的最终目标是找到并到达关卡出口。一旦与猎人发生碰撞，则游戏立即失败。

与AI斗智：AI 猎人并非全知全能。它会因丢失目标而进入概率搜索状态，玩家可借此机会误导其搜索方向，规划安全路线。

项目说明：

负责整体项目架构设计与模块划分，实现游戏主循环、状态管理与渲染管线。

实现玩家移动、掩体交互与碰撞检测等核心游戏机制。

设计并实现基于 FSM 的 AI 行为系统，涵盖巡逻（PATROL）、追逐（CHASE）、概率搜索（SEARCH）三种状态。

开发基于概率热度图（Belief Map）的智能搜索算法，模拟 AI 对玩家位置的动态推断与决策过程。

技术实现：

使用 SDL2 库实现跨平台的图形渲染、输入处理与音频管理；

基于有限状态机（FSM）构建 AI 行为逻辑框架，实现状态间的条件转换与平滑过渡；

实现视线检测系统，结合掩体判定机制，构成 AI 感知与玩家隐藏的核心规则；

采用概率热度图（Belief Array）模拟 AI 的“信念”，运用扩散（预测）与贝叶斯更新（证据融合）算法动态推断玩家最可能的位置；

在搜索行为中引入基于概率的加权随机目标选择，赋予 AI 合理的随机性与不可预测性，避免模式化巡逻或死守出口；

采用模块化代码结构，清晰分离游戏逻辑、AI 系统、渲染引擎与用户界面，保障代码可维护性与可扩展性。
